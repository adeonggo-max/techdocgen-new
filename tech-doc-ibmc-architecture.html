03

Data Flow Pipeline

1

User Input

User provides source type, LLM provider selection, and configuration options through Web UI or CLI.

File/Folder/Git LLM Selection Config Options
2

Source Reading Phase

Appropriate reader discovers, filters, and reads source files with language detection.

FileReader FolderReader GitReader
3

Parsing Phase

Language-specific parser extracts structure using Tree-sitter AST parsing for each file.

Classes Methods Imports Comments
4

Dependency Analysis

Optional phase that analyzes file dependencies, builds graphs, and detects circular dependencies.

Build Graph Detect Cycles Find Orphans
5

Documentation Generation

LLM provider generates natural language documentation from parsed information for each file.

Prompt Preparation LLM API Call Response Processing
6

Aggregation Phase

Combine all file documentation, add dependency maps and sequence diagrams to generate final output.

Merge Docs Add Diagrams Generate TOC
7

Output Phase

Save final documentation as Markdown and optionally generate PDF with Confluence-style formatting.

Markdown PDF Dependency Export
